PASO 1️⃣ - Crear Proyecto en Supabase

Configurar Base de Datos (5 min)

1. Crear cuenta

  1. Ve a https://supabase.com
  2. Click en Sign Up
  3. Usa tu email: josefraige@gmail.com
  4. Verifica tu correo

2. Crear nuevo proyecto

  1. Dashboard → New Project
  2. Nombre: trakwerk-parques
  3. Región: North America (us-east-1)
  4. Password: TuContraseña123Fuerte!
  5. Click Create new project
  6. ⏳ Espera 2-3 minutos a que se cree...

3. Copiar credenciales

  1. Ve a Settings → API
  2. Copia Project URL
  3. Copia anon public (bajo "Supabase key")
  4. Guarda en un archivo temporal

4. Crear tablas (copiar y pegar en SQL Editor)

  1. Ve a SQL Editor
  2. Click New Query
  3. Copia este SQL:
-- Tabla de parques CREATE TABLE parques ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), nombre TEXT NOT NULL, ubicacion TEXT, total_lotes INTEGER, lotes_disponibles INTEGER, created_at TIMESTAMP DEFAULT now() ); -- Tabla de lotes CREATE TABLE lotes ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), parque_id UUID REFERENCES parques(id), numero TEXT NOT NULL, metros DECIMAL, precio_por_metro DECIMAL, estado TEXT DEFAULT 'disponible', created_at TIMESTAMP DEFAULT now() ); -- Tabla de cotizaciones CREATE TABLE cotizaciones ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), nombre TEXT NOT NULL, telefono TEXT NOT NULL, parque TEXT NOT NULL, lote TEXT NOT NULL, created_at TIMESTAMP DEFAULT now() );
  1. Click Run (botón azul)
  2. ✅ Tablas creadas
PASO 2️⃣ - Subir a GitHub

Crear Repositorio (3 min)

1. Si no tienes GitHub

  1. Ve a https://github.com
  2. Click Sign up
  3. Usa email: josefraige@gmail.com
  4. Completa el registro

2. Crear nuevo repositorio

  1. Dashboard → New repository
  2. Nombre: trakwerk-parques
  3. Descripción: Aplicación de venta de lotes industriales
  4. Click Create repository

3. Copiar comando push

En la pantalla de tu nuevo repo, busca la sección "push an existing repository from the command line" y ejecuta en PowerShell:

cd C:\Users\Usuario\trakwerk-parques git remote add origin https://github.com/TU_USUARIO/trakwerk-parques.git git branch -M main git push -u origin main

⚠️ Reemplaza TU_USUARIO con tu usuario de GitHub

PASO 3️⃣ - Deploy en Vercel

Deploy Automático (2 min)

1. Conectar Vercel a GitHub

  1. Ve a https://vercel.com
  2. Click Sign up
  3. Selecciona Continue with GitHub
  4. Autoriza Vercel

2. Importar proyecto

  1. Dashboard → Add New... → Project
  2. Selecciona trakwerk-parques
  3. Click Import

3. Configurar variables de entorno

  1. Ve a Environment Variables
  2. Añade dos variables:
NEXT_PUBLIC_SUPABASE_URL = [PEGA TU URL DE SUPABASE] NEXT_PUBLIC_SUPABASE_ANON_KEY = [PEGA TU ANON KEY]
  1. Click Deploy
  2. ⏳ Espera 2-3 minutos
  3. ✅ Tu app estará en: https://trakwerk-parques.vercel.app
PASO 4️⃣ - Conectar Dominio

Agregar trakwerk.com.mx (2 min)

1. En Vercel

  1. Proyecto → Settings → Domains
  2. Click Add Domain
  3. Escribe: parques.trakwerk.com.mx
  4. Vercel te dará un CNAME

2. En tu proveedor DNS (GoDaddy, Namecheap, etc.)

  1. Inicia sesión en tu proveedor de dominios
  2. Ve a DNS Settings / Manage DNS
  3. Crea un nuevo CNAME record:
Nombre: parques Tipo: CNAME Valor: cname.vercel-dns.com TTL: 3600
  1. Guarda los cambios
  2. ⏳ Espera 15-30 minutos
  3. ✅ Tu app estará en: https://parques.trakwerk.com.mx
✅ Checklist Final

Verifica que todo funciona

⚠️ Si algo no funciona

  1. Deploy falla: Ve a Vercel → Deployments → mira los logs (tab "Build")
  2. Login no funciona: Abre DevTools (F12) → Console y busca errores
  3. Variables de entorno: Recuerda agregar el prefijo NEXT_PUBLIC_
  4. Dominio no apunta: Espera 30 min después de cambiar DNS
